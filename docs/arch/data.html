Data philosophy

The architecture is moving toward event-sourced learning data: mathAnswerEvents is the canonical record, while itemStates, attempts, multFactStats, and quizSessions are caches or compatibility views.

Good design choice: after sync, derived tables are rebuilt from the union of events, reducing stale aggregate conflicts across devices.

Wrong answer

Stored as reviewGrade='again'; retry stays on same question. First wrong attempt can add misconception tags.

Slow correct

Correct answers can become hard, not again, so they do not become false FSRS failures.

Retry

Retry events are saved for stats/history but do not update itemStates or FSRS.

Mastery status layers

Item state
new → learning → developing → strong → mastered
→
Skill summary
new / needs practice / review due / strong / mastered

Item mastery comes from accuracy and FSRS stability. Skill mastery comes from first-attempt events, due item counts, number of distinct items, and thresholds.

Important: A skill should not be marked mastered from repeated drilling of a single item. The skill engine requires multiple distinct items for mastery.

Sync architecture

Auth

Google Identity Services token client is preloaded; tokens stay in memory only.

Storage

Drive file name is mathfan-data.json in the private appDataFolder.

Conflict model

Events are merged by ID; derived tables are recomputed after merge.

PWA update flow

Vite PWA is configured with Workbox precaching, skipWaiting, and clientsClaim. main.tsx listens for controllerchange and reloads after a new service worker activates, except on the first install.

Take care: when debugging “old UI” issues, verify service worker state, app build version, base path, and cache update behavior.

AI/audio state

AI key and model live in localStorage, not the profile or sync snapshot. Audio uses browser speech synthesis and normalizes math expressions before speaking.

Take care: localStorage API keys are convenient but not high-security. A sign-in-gated serverless proxy is a better public-scale option.
