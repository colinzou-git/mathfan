Practice flow

1. App chooses practice screen

App.tsx saves return destination and sends SessionConfig to PracticeScreen.

2. Session queue is built

usePracticeSession generates dynamic items or selects specific/daily-review items.

3. Student answers

answerChecker validates input and maps correctness/latency to again/hard/good/easy.

4. First attempt schedules

applyReview calls the FSRS adapter for durable item state and next due date.

5. Event is written

recordPracticeAnswer stores answer event, item state, and compatibility attempt log in one Dexie transaction.

6. UI advances

Auto-advance waits for correct-answer speech to finish, preventing the next question from cutting off the answer audio.

Mastery-map flow

1. Load events + states

Map page loads mathAnswerEvents and itemStates.

2. Reconstruct item metadata

makeItemFromId resolves dynamic IDs into typed items.

3. Infer skill IDs

skillMapping maps item types/tags to Grade 3 skill nodes.

4. Derive summaries

skillMasteryEngine calculates attempts, accuracy, due count, status, and misconception tags.

5. Plan today

todayPlanEngine picks warmup, focus, review, and prerequisite advisory.
